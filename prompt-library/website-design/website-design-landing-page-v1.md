# Landing Page Design Brief

Title: Landing Page Design Brief
Category: website-design
Use case: Landing page
Version: v1
Tags: website-design, landing-page, conversion, responsive, product-marketing

## Prompt

Design a production-quality landing page for `{product_name}`, a `{product_type}` for `{target_audience}`. The page should communicate the core offer clearly, build trust quickly, and guide visitors toward `{primary_action}`.

Include a first viewport with a direct headline, supporting copy, primary and secondary actions, and a strong visual direction. Then define sections for key benefits, proof, workflow or features, pricing or plan teaser if relevant, FAQ, and final call to action. Keep the design practical, responsive, accessible, and easy for a developer to implement.

Avoid generic startup language. Use concrete details from the inputs and make the page feel specific to the product.

## Input variables

- `product_name`: Name of the product or offer.
- `product_type`: Short description of what it is.
- `target_audience`: Who the page is for.
- `primary_action`: Main action visitors should take.
- `brand_tone`: Desired voice and visual tone.
- `key_benefits`: 3 to 5 product benefits.
- `proof_points`: Testimonials, metrics, logos, or credibility signals.

## Expected output

A structured landing page brief with page goal, audience, section-by-section layout, suggested copy, visual direction, interaction notes, and accessibility considerations.

## Example input

`product_name`: LaunchPad  
`product_type`: planning tool for solo founders  
`target_audience`: founders validating a software idea  
`primary_action`: start a free workspace  
`brand_tone`: calm, focused, practical  
`key_benefits`: clarify scope, organize tasks, track validation, reduce busywork  
`proof_points`: 1,200 beta users, founder testimonials

## Example output

A calm landing page with a hero headline like "Turn a rough idea into a testable launch plan." Sections include founder pain points, a three-step planning workflow, product screenshots, beta-user quotes, pricing teaser, FAQ, and a focused final CTA to start a free workspace.
