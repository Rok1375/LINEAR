# Pricing Page Design Brief

Title: Pricing Page Design Brief
Category: website-design
Use case: Pricing page
Version: v1
Tags: website-design, pricing, conversion, saas, plans

## Prompt

Create a pricing page for `{product_name}` that helps `{target_audience}` choose the right plan with confidence. The page should make differences between plans clear, reduce hesitation, and support `{primary_action}`.

Include plan structure, comparison table guidance, trust signals, FAQ topics, upgrade prompts, and responsive layout. Make the recommendation logic clear without pushing every visitor toward the highest tier.

## Input variables

- `product_name`: Product name.
- `target_audience`: Buyer or user segment.
- `plans`: Plan names, prices, and intended users.
- `primary_action`: Desired pricing-page action.
- `key_limitations`: Usage limits or feature gates.
- `trust_points`: Refund policy, support promise, security notes, or customer proof.

## Expected output

A pricing page brief with plan positioning, comparison structure, recommended-plan treatment, supporting copy, FAQ topics, and UX notes.

## Example input

`product_name`: PromptVault  
`target_audience`: small creative teams  
`plans`: Free, Pro at $12/user, Team at $29/user  
`primary_action`: start free trial  
`key_limitations`: prompt count, shared folders, optimization runs  
`trust_points`: cancel anytime, encrypted storage, priority support

## Example output

A pricing page with three plan columns, Pro highlighted for solo professionals, Team positioned for shared libraries and approvals, a compact feature comparison, security reassurance, and FAQs about prompt limits and cancellation.
