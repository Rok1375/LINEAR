## Ethical Crawling Notes

- Prefer official APIs, exports, feeds, or existing open datasets before crawling pages.
- Never crawl paywalled, private, logged-in, credentialed, or explicitly blocked content.
- Check `robots.txt`, terms of service, rate limits, sitemap guidance, and target-site load before every crawl.
- Use a clear user agent/contact when running a crawler under your own infrastructure.
- Cap concurrency, retries, total pages, runtime, and spend before starting any crawler or hosted scraping job.
- Treat community scrapers, public Actors, and MCP tools as untrusted until their permissions, code, output, and costs are reviewed.

## Recommended Prototype Shortlist

1. **OpenAI Agents SDK**: first OpenAI-native agent implementation candidate.
2. **LangGraph**: durable graph/state orchestration candidate for longer-running workflows.
3. **Pydantic AI**: type-safe Python agent candidate for smaller production services.
4. **n8n**: self-hostable/hosted automation candidate with human-review workflow affordances.
5. **Zapier MCP**: broad SaaS action layer candidate, only with aggressively curated actions.
6. **Apify or Crawlee**: crawling/data extraction candidate depending on hosted versus local needs.
7. **Cloudflare Browser Rendering**: managed browser/crawl candidate for permitted web rendering at scale.
8. **Common Crawl**: live-crawl avoidance candidate for broad public web research.
